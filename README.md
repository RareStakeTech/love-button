# ReddID Love Button v2

Chrome extension (Manifest V3) that lets you look up ReddID @handles and tip creators with RDD directly from your browser.

## Features

- **Popup search** — type any ReddID @handle to retrieve the linked RDD address
- **Auto-detection** — when you visit a creator's Twitter/X, Reddit, or YouTube page the extension detects them automatically and shows a banner
- **In-page tip buttons** — injects a "🔴 Tip with RDD" button next to Follow/Subscribe on supported platforms
- **Copy to clipboard** — one-click copy of the RDD address ready to paste into any wallet
- **Open tip page** — jumps straight to the creator's public tip page on redd.love
- **5-minute cache** — lookups are cached in local storage; clears on settings save or manual purge

## Supported platforms

| Platform | Trigger | Injected button location |
|----------|---------|--------------------------|
| Twitter/X | `/username` profile pages | Next to Follow button |
| Reddit | `/user/username` and `/u/username` pages | Next to profile actions |
| YouTube | `/@channelname`, `/channel/`, `/c/`, `/user/` pages | Next to Subscribe button |

## Install (development / unpacked)

1. Clone or copy this folder somewhere permanent on your machine.
2. Open Chrome and navigate to `chrome://extensions/`.
3. Enable **Developer mode** (toggle, top-right).
4. Click **Load unpacked** and select this folder (`love-button/`).
5. The REDD icon appears in your toolbar. Pin it for quick access.

> **Note**: Icons were generated by `gen-icons.js`. If you replace them, keep the same file names (`icon16.png`, `icon48.png`, `icon128.png`) and update `manifest.json` if needed.

## Settings

Click the REDD icon → **Settings** (bottom-left of the popup) to open the options page.

| Setting | Default | Description |
|---------|---------|-------------|
| API base URL | `https://redd.love` | Root URL of the ReddID Next instance to query |

Change the API base to point at a local dev server (`http://localhost:3000`) while developing.

### API contract

The extension expects the ReddID Next API to expose:

```
GET /api/identities/{handle}
→ { identity: { handle, displayName, bio, rddAddress, socialProofs, ... } }
   or 404

GET /api/identities/by-social?platform={platform}&username={username}
→ { identity: { ... } }
   or 404
```

### Message types (popup/content → background)

| Type | Payload | Response |
|------|---------|----------|
| `LOOKUP_HANDLE` | `{ handle }` | `{ identity }` or `{}` |
| `LOOKUP_SOCIAL` | `{ platform, username }` | `{ identity }` or `{}` |
| `GET_API_BASE` | — | `{ base }` |
| `CLEAR_CACHE` | — | — |

### Running a local ReddID Next instance

```bash
cd reddid-web
npm install
npm run dev
# Runs on http://localhost:3000
```

Then in extension settings, set API base to `http://localhost:3000`.

## Privacy

- No analytics, no telemetry.
- Handle lookups hit only the configured API base (default: redd.love).
- The extension requests `activeTab` (not `<all_urls>`) — tab URL is only read when the popup is open.
- Host permissions are restricted to twitter.com, x.com, reddit.com, and youtube.com for content script injection.
- No data is sent anywhere except the ReddID Next API for handle resolution.
